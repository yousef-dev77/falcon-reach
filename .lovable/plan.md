# خطة بناء وحدة نقاط البيع (POS)

## الهدف
نظام POS احترافي شبيه بـ Odoo POS، يعمل على الشاشات اللمسية ويتكامل مع المخزون والمحاسبة.

## المكونات

### 1. قاعدة البيانات (Migration)
- `pos_configs` — إعدادات كل نقطة بيع (الفرع، المستودع، الصندوق، الحساب البنكي، السماح بالخصم، الضريبة الافتراضية)
- `pos_sessions` — جلسات الكاشير (فتح/إغلاق، رصيد افتتاحي/ختامي، الفروقات)
- `pos_orders` — فواتير POS (رقم، عميل اختياري، الإجمالي، الضريبة، الخصم، الحالة: draft/paid/invoiced/cancelled)
- `pos_order_lines` — أصناف الفاتورة
- `pos_payments` — مدفوعات الفاتورة (نقدي/شبكة/تحويل) مع دعم المدفوعات المتعددة
- `pos_cash_movements` — حركات النقد داخل الجلسة (إيداع/سحب)

### 2. الدوال (Functions)
- `open_pos_session` — فتح جلسة وإقفال الجلسات السابقة المفتوحة
- `close_pos_session` — إقفال الجلسة + توليد قيد محاسبي (مدين: صندوق+بنك / دائن: إيرادات+VAT) + خصم المخزون
- `pay_pos_order` — تأكيد الدفع وتحويل الحالة إلى paid
- `create_invoice_from_pos_order` — إنشاء فاتورة مبيعات رسمية من طلب POS (للعملاء المسجلين)

### 3. الواجهات (Pages)
- `src/pages/pos/POSConfigs.tsx` — إدارة نقاط البيع
- `src/pages/pos/POSSessions.tsx` — قائمة الجلسات (فتح/إغلاق/تقارير)
- `src/pages/pos/POSTerminal.tsx` — **شاشة الكاشير الرئيسية** (Grid منتجات + Cart + لوحة دفع touch-friendly)
- `src/pages/pos/POSOrders.tsx` — سجل الفواتير
- `src/pages/pos/POSReports.tsx` — تقرير المبيعات اليومية + تقرير X/Z

### 4. التكامل
- خصم تلقائي من المخزون عند الدفع (inventory_movements)
- قيد محاسبي تلقائي عند إغلاق الجلسة
- ترقيم تلقائي: `POS-[BranchCode]-XXXXX`
- دعم RTL والخط Cairo

### 5. التحديثات الجانبية
- `App.tsx` — إضافة المسارات
- `AppSidebar.tsx` — قسم "نقاط البيع" مع الأيقونات
- `types.ts` — تحديث تلقائي بعد الـ migration
